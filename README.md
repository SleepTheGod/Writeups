# Security Policy

## Reporting a Vulnerability

If you discover any vulnerability in any of the projects documented here:

1. **Do NOT publicly disclose it**.
2. Contact me directly at: [your-email@example.com] or open a **private GitHub security report**.
3. Provide:
   - A detailed description of the issue
   - Steps to reproduce
   - Affected writeup or software
   - Any proof-of-concept code (if safe to share)

I will acknowledge receipt of your report within **48 hours** and provide updates on resolution.

---

## Supported Versions

This repository primarily documents security research and proof-of-concepts. There are no software versions being maintained; treat these writeups as reference material.

---

## Acknowledgements

I follow responsible disclosure guidelines. Thank you for helping maintain a safe and educational environment.

# Contributing to Security Writeups

Thank you for your interest in contributing! This repository is primarily a personal collection of writeups, but contributions are welcome under the following guidelines:

## What You Can Contribute

- Corrections or clarifications to existing writeups
- Additional writeups that are educational and safe to share
- Suggestions for formatting, structure, or readability

## Guidelines

1. Ensure writeups are safe to share publicly (no active exploits against live systems without permission).
2. Follow the existing Markdown style.
3. Include:
   - Target / Challenge Description
   - Tools used
   - Exploit steps / PoC
   - Mitigation / Lessons learned

## How to Contribute

1. Fork the repository
2. Create a branch for your writeup or changes
   ```bash
   git checkout -b my-writeup
````

3. Add your file or edits
4. Submit a Pull Request describing your contribution

---
